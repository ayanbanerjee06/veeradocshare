🖥️1. Web Frontend — PWA14 technologies

The web client is a React-based Progressive Web App served through a Module Federation shell. Each feature area is an independently deployed microfrontend remote that the shell lazy-loads at runtime.

TechnologyVersionUsage / Purpose
React 18V1Core UI framework. Concurrent rendering, Suspense for lazy MFE loading, and automatic batching. Used in the shell and every MFE.
TypeScriptV1Strict typing across all frontend code. Shared type packages published from the monorepo are consumed by both frontend and backend services.
ViteV1Build tool for each individual MFE. Provides instant HMR during development and optimised ESM bundles for production.
Webpack 5 Module FederationV1Microfrontend host/remote wiring. The shell fetches and mounts MFE bundles from separate deployment URLs at runtime, enabling independent team deploys.
React Router v6V1Client-side routing inside the shell. Each MFE registers its own route subtree, enabling deep-link support per feature domain.
ZustandV1Lightweight global state shared across MFEs via the shell. Holds auth user context, current match, notification queue, and wallet balance.
TanStack QueryV1Server state management — handles caching, background refetch, optimistic updates, and offline support. Eliminates boilerplate fetch/loading/error state.
Tailwind CSSV1Utility-first CSS framework. Shared token configuration lives in a design system package; all MFEs import the same design tokens for consistent UI.
WorkboxV1Service Worker toolkit for PWA functionality. Network-first strategy for API calls, cache-first for static assets, background sync for offline actions.
Socket.io ClientV1Real-time WebSocket client. Handles live chat messages, typing indicators, match notifications, safety check-ins, and live trust score updates.
WebRTCV1Peer-to-peer audio and video calls. The shell manages the signalling channel via Socket.io. STUN/TURN relay provided by Twilio for NAT traversal.
StorybookV1Design system component library and visual testing environment. All shared components documented and visually tested before use in MFEs.
VitestV1Unit and integration testing framework. Vite-native for fast execution. Used with React Testing Library for component behaviour tests.
PlaywrightV1End-to-end browser testing. Covers full user flows: sign-up → swipe → match → chat → book an experience.
📱2. Mobile App — Ionic + Capacitor8 technologies

The mobile app shares approximately 85% of its codebase with the web MFEs. Ionic React provides mobile-optimised UI components, while Capacitor bridges to native device APIs without Cordova.

TechnologyVersionUsage / Purpose
Ionic ReactV1Mobile-optimised UI components (sheets, tabs, swipe gestures) that match native look and feel on iOS and Android while reusing the React codebase.
CapacitorV1Native runtime bridge. Wraps the React PWA in a native shell and provides a plugin system for device API access without Cordova dependencies.
Capacitor CameraV1Profile photo and video capture. Also provides the camera feed for liveness detection during identity verification.
Capacitor GeolocationV1Used for nearby profile discovery, date location sharing, real-time safety location tracking, and event check-in.
Capacitor Push NotificationsV1FCM (Android) and APNs (iOS) native push. Handled in the shell and routed to the appropriate MFE for display.
Capacitor BiometricsV2Face ID and fingerprint authentication for fast re-auth flows and panic button trigger. Added as part of the V2 Female Safety Suite.
Capacitor FilesystemV1Offline media caching, QR event ticket storage, and downloaded gift card assets for use without connectivity.
Capacitor Screen ReaderV2Accessibility support for screen readers. Required for India market compliance and inclusive design.
⚙️3. Backend Services14 technologies

All backend services follow a NestJS-first pattern for consistency, dependency injection, and enforced module boundaries across a 20+ service estate. Python handles AI workloads; Go is introduced at V2 for the highest-throughput services.

TechnologyVersionUsage / Purpose
Node.js 20 LTSV1Primary runtime for Auth, Profile, Matching, Chat, Wallet, Gift, Experience, Notify, Trust Score, Event, Safety, Subscription, Couple, Merchant, and Marketplace services.
NestJSV1Structured Node.js framework. Dependency injection, guards, interceptors, and decorators enforce consistent service architecture across all Node.js services.
Fastify AdapterV1HTTP adapter inside NestJS. Approximately 2× faster throughput than the default Express adapter for high-volume API services.
Python 3.11V1Language for AI Svc, Concierge AI Svc, and ML Pipeline. Chosen for native LangChain, PyTorch, and scikit-learn integration.
FastAPIV1Python REST and async framework for AI services. Auto-generates OpenAPI documentation, includes Pydantic request validation, and supports async inference calls.
Go 1.22V2High-throughput rewrite of the Discovery and Matching services at V2 scale. ~10× the request throughput of Node.js for the hot swipe feed at 100k+ DAU.
GraphQL — Apollo ServerV1BFF aggregation layer. A single GraphQL query returns profile, match status, and chat state together, reducing mobile round trips significantly.
REST — OpenAPI 3V1All inter-service HTTP communication. Spec-first: swagger-cli generates type-safe client SDKs from the OpenAPI spec for each consuming service.
Passport.jsV1OAuth strategy handling within the Auth service — Google, Facebook, Apple, and LinkedIn sign-in are each implemented as Passport strategy modules.
BullMQV1Redis-backed job queues for background workers: photo moderation, gift dispatch, subscription renewal, trust score recalculation, and ML inference batching.
Prisma ORMV1Type-safe PostgreSQL client used in NestJS services. Schema-first migrations and a generated query client eliminate raw SQL and type mismatches.
MongooseV1MongoDB ODM for Profile, Experience, and Marketplace services. Schema-flexible for deeply nested lifestyle, preference, and interest data.
class-validatorV1DTO validation decorators inside NestJS. Input sanitisation and schema validation applied to all incoming API payloads.
Helmet.jsV1HTTP security header middleware applied to all services. Protects against XSS, clickjacking, and MIME sniffing attacks.
🗄️4. Data & Storage9 technologies

Each microservice owns its own data store — no shared databases. PostgreSQL is the default for transactional data; MongoDB handles schema-flexible document data; Redis serves caching and queuing; DigitalOcean Spaces provides object storage.

TechnologyVersionUsage / Purpose
DO Managed PostgreSQLV1Primary relational database. Used by Auth, Matching, Wallet, Gift, Event, Trust Score, Subscription, Safety, Couple, Merchant, and Verification services. Standby replicas and daily automated backups.
MongoDBV1Document store for Profile, Experience, and Marketplace services. Schema flexibility for nested lifestyle preferences, interest tags, and catalogue item attributes.
DO Managed RedisV1Multi-purpose in-memory store: API response caching, session tokens, BullMQ job queues, match candidate queues, gift redemption code lookup, and rate limiting counters.
DigitalOcean SpacesV1S3-compatible object storage for profile photos, intro videos, voice notes, gift images, and event banners. Built-in CDN edge ensures low-latency media delivery globally.
DO Managed OpenSearchV2Full-text and vector search for profile discovery, event search, gift catalogue browse, and experience search. Added at V2 when search quality becomes critical.
PineconeV1Dedicated vector database. Stores personality and interest embeddings generated by OpenAI. Powers semantic similarity matching beyond keyword-based filters.
ClickHouseV3Columnar analytics database. Handles partner revenue dashboards, conversion funnel analysis, behavioural cohort queries, and high-volume event ingestion. Introduced at V3.
SharpV1Node.js image processing library. Auto-resizes profile photos to multiple resolutions on upload. Runs as a BullMQ background worker to avoid blocking API responses.
FFmpegV1Video and audio transcoding for profile intro videos and voice notes. Runs as a sidecar container in Kubernetes to transcode media asynchronously after upload.
🤖5. AI & Machine Learning11 technologies

AI is a core differentiator across three dimensions: profile coaching (V1), trust and safety intelligence (V2), and proprietary compatibility matching (V3). The stack is LLM-gateway first, with custom ML added incrementally as proprietary data accumulates.

TechnologyVersionUsage / Purpose
OpenAI GPT-4oV1Primary LLM for profile bio generation, icebreaker suggestions, conversation starters, relationship coaching responses, conflict guidance, and date planning.
Claude 3.5 SonnetV1Fallback and specialist LLM for long-context relationship advice and nuanced coaching exchanges where GPT-4o context limits or tone are insufficient.
LangChain — PythonV1LLM orchestration framework. Manages RAG chains, tool calling, prompt templates, and conversation memory across all AI service endpoints.
LangGraphV2Agentic workflow engine for multi-step AI Concierge tasks: iterative date planning, personalised gift recommendation loops, and coaching sessions that retain state across turns.
OpenAI EmbeddingsV1text-embedding-3-large model. Encodes profiles, interest tags, and messages into vectors stored in Pinecone for semantic similarity search and match scoring.
Hugging Face TransformersV2Fine-tuned models for: fraud detection NLP classifier, message toxicity filter, and profile quality scorer — all trained on app-specific labelled data at V2 scale.
scikit-learnV2Trust Score feature engineering — logistic regression and gradient boosting models trained on verification level, response rate, event participation, report history, and profile completeness.
PyTorchV3Custom compatibility model training on proprietary behavioural data. Predicts relationship likelihood from personality vectors, communication patterns, and long-term goal alignment.
MLflowV3ML experiment tracking, model versioning, and deployment registry for the V3 compatibility model. Ensures reproducibility and safe rollout of new model versions.
Celery + RedisV2Async ML inference task queue. Runs batch trust score recalculation, nightly embedding generation for new profiles, and scheduled model scoring jobs.
DO GPU DropletsV3GPU compute instances (H100) for scheduled model training runs triggered by the ML Pipeline via Celery. Spun up for training and torn down after to minimise cost.
⚡6. Real-time & Communication9 technologies

Real-time is central to the product — chat, calls, safety check-ins, and match notifications all require low-latency delivery. The stack uses WebSocket for bidirectional messaging, WebRTC for media, and Kafka for async inter-service events.

TechnologyVersionUsage / Purpose
Socket.ioV1Bidirectional WebSocket server and client. Handles: chat messages, typing indicators, match notifications, live trust score updates, safety check-in pings, and real-time swipe feed events.
WebRTCV1Peer-to-peer audio and video calls. The shell manages signalling via the Socket.io connection. The Chat service negotiates WebRTC session parameters.
Twilio TURNV1STUN/TURN relay server for WebRTC. Essential for NAT traversal on Indian mobile networks and corporate firewalls where direct peer connections frequently fail.
Firebase Cloud MessagingV1Android push notification delivery: match alerts, new messages, safety check-in nudges, event reminders, and gift received notifications.
Apple Push Notification ServiceV1iOS push delivery. Routed through the Capacitor Push Notifications plugin and dispatched by the Notify microservice.
SendGridV1Transactional email: welcome onboarding, daily match digest, gift received confirmation, event reminders, subscription receipts, and trust badge earned notifications.
Twilio SMSV1SMS OTP for phone number verification and two-factor authentication. Also sends panic button SMS alerts to the user's registered emergency contacts.
DO Managed KafkaV2Async event bus between microservices. Key topics: match.created, message.sent, gift.redeemed, trust.updated, event.checkedin, subscription.renewed. Decouples services and enables event replay.
Daily.coV2Fallback video call infrastructure when WebRTC quality degrades on poor connections. Also used for Virtual Date Rooms available to Platinum members.
🏗️7. Infrastructure & DevOps16 technologies

All infrastructure runs on DigitalOcean. Kubernetes (DOKS) orchestrates all services; Terraform codifies every resource; GitHub Actions delivers continuous deployment. Cloudflare sits at the edge for CDN, WAF, and DDoS protection.

TechnologyVersionUsage / Purpose
DOKSV1DigitalOcean Kubernetes Service. Separate auto-scaling node pools per service tier: general API services, ML inference workers, and database proxies.
HelmV1Kubernetes package manager. Each microservice has a Helm chart covering deployment spec, HPA (autoscaling), ingress rules, config maps, and Vault secret references.
DockerV1All services and MFE builds are containerised using multi-stage Dockerfiles to minimise image size. Images stored in the DO Container Registry.
DO Container RegistryV1Private Docker image registry integrated natively with DOKS. Eliminates dependency on external registries and simplifies pull secret management.
TerraformV1Infrastructure as code for all DigitalOcean resources: DOKS clusters, Spaces buckets, managed databases, firewall rules, and load balancers.
GitHub ActionsV1CI/CD pipeline. On merge to main: lint → unit test → Docker build → push to DO Registry → Helm upgrade on DOKS staging → smoke test → promote to production.
CloudflareV1CDN, WAF, DDoS protection, DNS, and edge-level rate limiting. First line of defence before traffic reaches the API Gateway on DigitalOcean infrastructure.
Nginx IngressV1Kubernetes ingress controller. Routes external traffic to the correct service pods. Handles TLS termination using certificates issued by cert-manager.
cert-managerV1Automatic TLS certificate provisioning and renewal via Let's Encrypt for all service endpoints. Certificates stored as Kubernetes secrets.
HashiCorp VaultV1Secrets management. API keys, database credentials, and JWT signing secrets are injected into pods at runtime via the Vault Agent sidecar. No secrets in source control.
DatadogV1APM, distributed tracing, log aggregation, custom dashboards, and PagerDuty alerting. The Datadog agent runs as a DaemonSet on all DOKS nodes.
OpenTelemetryV2Distributed tracing SDK instrumented in every service. Traces flow across Auth → Matching → Chat → Notify service boundaries. Exports spans to Datadog.
SentryV1Frontend and backend error tracking. Provides stack traces, release health monitoring, and user-facing error rate alerting for React MFEs and Node.js services.
k6V1Load testing tool. Run against staging before every major release to validate throughput on the swipe feed, match endpoint, chat service, and payment flows.
SnykV1Dependency vulnerability scanning integrated into the GitHub Actions CI pipeline. Blocks deployments when critical CVEs are detected in npm or Python dependencies.
DO Multi-region DOKSV3Secondary Kubernetes cluster in a second DigitalOcean region (Bangalore + Singapore). Active-passive for disaster recovery; active-active at V3 scale for geographic latency optimisation.
🔌8. Third-party Integrations13 technologies

External services are used for identity verification, payments, social authentication, mapping, gifting fulfilment, and analytics. Each integration is wrapped in an internal adapter service to isolate the rest of the system from provider API changes.

TechnologyVersionUsage / Purpose
OnfidoV1Primary KYC provider. Performs government ID OCR, face match against the ID photo, and liveness video detection. The Onfido SDK is embedded inside the Verification MFE.
JumioV1Backup KYC provider. Used as a fallback when Onfido is unavailable and for specific regional document types where Onfido accuracy is lower.
RazorpayV1Primary payment gateway for India. Supports UPI, cards, net banking, and EMI. Used for wallet top-up, subscription billing, event ticket purchase, and gift payments.
StripeV1International payment fallback for non-India users. Handles card processing and subscription lifecycle (dunning, proration) for international accounts.
LinkedIn APIV1Professional verification flow. Employer name, job title, and LinkedIn profile URL validated via OAuth inside the Verification MFE to earn the professional verification badge.
Facebook Login / Graph APIV1Social sign-in and profile photo import. The Graph API is also used for Instagram account linking to strengthen identity trust signals.
Google Sign-InV1OAuth sign-in on web and mobile via the Capacitor Google Auth plugin. Also provides the identity token used for Calendar API access.
Google Maps APIV1Nearby profile radius calculation, date location sharing map display, event venue rendering, and experience location lookup throughout the app.
Google Calendar APIV2Event RSVP synchronisation. When a user RSVPs to an event, it is automatically added to their Google Calendar. Introduced with the V2 Events platform.
TwilioV1Three products: SMS for OTP and panic button alerts, Voice for call fallback, and the TURN relay service for WebRTC NAT traversal.
FNP API — Ferns N PetalsV3Physical gift fulfilment partner. API enables in-app ordering of flowers, cakes, and personalised gifts with order status tracking. Part of the V3 Full Marketplace.
IGP APIV3Secondary gift fulfilment partner. Covers chocolates, jewellery, and luxury gift categories not served by FNP. Also part of the V3 Full Marketplace.
HotjarV2Session recordings and click heatmaps on the web app. Used by the product team for UX optimisation and conversion improvement during the V2 engagement push.
